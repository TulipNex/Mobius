3. Variabel Global & Config

Database: Gunakan global.db.data untuk akses manipulasi database (contoh: global.db.data.users[m.sender], global.db.data.chats[m.chat], atau global.db.data.settings[conn.user.jid]).

Respon Default: Gunakan variabel global seperti global.wait untuk memunculkan pesan loading saat proses berlangsung, dan global.eror untuk penanganan pesan error sistem.

4. Library, Helper & Pemrosesan API

Dalam menangani networking dan data pihak ketiga (berdasarkan keahlian API Reverse Engineering & Scraping):

Networking & Scraping: Gunakan node-fetch (untuk hit API dasar), serta paduan axios dan cheerio (untuk scraping HTML/Web).

Header Spoofing: Jika membuat scraper API tersembunyi (hidden API), SELALU sediakan konfigurasi HEADERS yang valid (termasuk User-Agent, Referer, dan Client Hints seperti sec-ch-ua) untuk menghindari pemblokiran anti-bot (WAF/Cloudflare).

Penanganan Media: Gunakan helper seperti require('../lib/uploadFile') atau uploadImage jika perlu mengunggah media ke server pihak ketiga (contoh: Catbox, Telegra.ph). Gunakan teknik Buffer atau Stream (fs.createReadStream) dengan header multipart/form-data yang benar.

Messaging: Gunakan m.reply('teks') untuk balasan teks sederhana dan conn.sendFile(m.chat, buffer/url, 'nama.ext', 'caption', m) untuk mengirim media (gambar/video/dokumen).

5. Prinsip Pembuatan Kode

Modular & Mandiri: Pastikan setiap plugin bisa berdiri sendiri dan siap diletakkan langsung di dalam folder /plugins, sedangkan scraper akan diletakkan di folder /lib

Validasi Input: Selalu cek ketersediaan parameter (contoh: if (!text) return m.reply(\Masukkan input!\nContoh: ${usedPrefix + command} query`)`).

Keseimbangan Fitur:

Downloader: Wajib menggunakan blok try-catch karena API pihak ketiga sering fluktuatif/mudah mati. Jika error, fallback ke pesan error yang mudah dipahami (user-friendly).

Internet/Tools: Sajikan data hasil scraping atau API secara rapi. Gunakan format list atau blockquote agar enak dibaca di WhatsApp.

RPG/Economy: Gunakan dan sesuaikan variabel database yang sudah ada seperti money, limit, exp, level, dan inventory lainnya secara terstruktur dan proporsional.

Keamanan Fitur: Pasang flag yang sesuai seperti handler.limit = true (mengurangi limit user), handler.premium = true (khusus user premium), atau handler.group = true (hanya bisa di dalam grup).

Gaya Bahasa: Berikan solusi dengan pendekatan clean code. Sisipkan komentar penjelas yang singkat pada alur logika yang kompleks (seperti saat bypass API atau kalkulasi token). Fokus pada output copy-paste ready.
